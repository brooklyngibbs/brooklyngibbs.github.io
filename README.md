# brooklyngibbs.github.io

My portfolio. Lives at [brooklyngibbs.github.io](https://brooklyngibbs.github.io).

Made by hand — HTML and CSS, no framework, no build step. The home page is quiet on purpose; the case studies do the heavy lifting.

## What's in it

Three case studies, each making a different argument:

- **[Offline Crush](https://brooklyngibbs.github.io/offline-crush/)** — my newsletter on internet culture. 8K+ subscribers, no paid acquisition. *Proves I build deep cultural taste over time.*
- **[Whimbitz](https://brooklyngibbs.github.io/whimbitz/)** — TikTok-style app for AI-generated mini-games. Shipped on iOS and Android, end-to-end. *Proves I ship products across the stack.*
- **[internet.siren](https://brooklyngibbs.github.io/tiktok/)** — my TikTok account. 3.5K+ followers in a month, 70K saves on the top post. *Proves I move fast on platforms.*

Together they argue I'm the rare hybrid who does all three.

## How it's built

Hand-rolled. One stylesheet per page (shell + one per case study), static assets in `assets/`, native `<dialog>` for the lightbox modals, CSS scroll-snap for carousels. About 15 lines of vanilla JS total, all of it on the Whimbitz Promo section. Hosted on GitHub Pages.

## Aesthetic

White background. Editorial typography — Instrument Serif, Inter, DotGothic16, Jacquarda Bastarda 9. Project assets (carousels, mascots, app screenshots, brand marks) do the visual work. The shell is the frame; the work is the art.

References I keep returning to: [Laurel Schwulst](https://laurelschwulst.com/home/), [Bleed](https://bleed.com/work), [Tammy Taabassum](https://taamannae.dev/).

## Run it locally

```bash
python3 -m http.server 8000
```

Then visit `localhost:8000`.
